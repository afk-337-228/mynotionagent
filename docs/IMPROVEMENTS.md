# Идеи по улучшению Notion Telegram Agent

## Быстрые победы (1–2 часа)

### 1. Inline-кнопки под «последние заметки» (/last)
**Что:** После вывода списка последних заметок показывать под сообщением кнопки: «Выполнено», «Удалить», «В категорию…» для последней заметки (или нумерованные кнопки для каждой).
**Зачем:** Не нужно вручную писать «выполнено» или «удали последнюю» — один тап.
**Как:** `ReplyKeyboardMarkup` или `InlineKeyboardButton` с `callback_data=page_id:action` (done/delete). Обработчик `CallbackQueryHandler` вызывает существующие `handle_done` / `handle_delete` по `page_id`.

### 2. Конфиг через env
**Что:** Вынести в переменные окружения: `CONFIDENCE_THRESHOLD` (по умолчанию 0.6), при желании `RATE_LIMIT_REQUESTS` и `RATE_LIMIT_WINDOW_SEC` из `state.py`.
**Зачем:** Менять порог уверенности и лимиты без правки кода (разные окружения: прод/тест).

### 3. Команда /tasks или /today ✅
**Что:** Новая команда: показать только страницы из базы «Задачи на сегодня/завтра» за сегодня (или с Due Date = сегодня), например первые 10–15.
**Зачем:** Быстро видеть только сегодняшние задачи, не смешивая с остальными заметками из /last.
**Реализовано:** Команда `/today`, `NotionClient.get_tasks_due_today(limit=15)`.

### 4. Подтверждение перед удалением
**Что:** При «удали последнюю» / «удали заметку X» сначала ответ: «Удалить «Название»? Да / Нет» (inline-кнопки). По «Да» — вызвать `archive_page`.
**Зачем:** Меньше случайных удалений.
**Как:** Состояние в `state`: `pending_delete_page_id` + таймаут (например 60 сек). Или сразу inline-кнопки «Удалить» / «Отмена» под сообщением с превью заметки.

## Средний приоритет (полдня–день)

### 5. Поиск по заметкам ✅
**Что:** Фраза «найди заметки про молоко» или «поиск: крипта» — поиск по Notion (Search API или фильтр по title в своих базах) и ответ списком до 5–10 совпадений со ссылками.
**Зачем:** Не помнишь точную формулировку — нашёл по ключевым словам.
**Реализовано:** Парсер «найди заметки про X», «найди про X», «поиск X»; `NotionClient.search_pages(query, limit=10)` через Notion Search API с фильтром по нашим базам.

### 6. Умные даты в задачах ✅
**Что:** При создании задачи извлекать из текста «завтра купить молоко», «в пятницу созвон» и подставлять Due Date без явного указания категории.
**Зачем:** Один запрос вместо «запиши в задачи: купить молоко» + ручное изменение даты в Notion.
**Реализовано:** В промпте и ответе ИИ поле `due_date_relative` (today/tomorrow/day_after_tomorrow/monday..sunday); `_resolve_due_date_from_intent()` переводит в YYYY-MM-DD и передаётся в `_save_note_and_respond(..., due_date=...)`.

### 7. Валидация секрета webhook (Telegram)
**Что:** В `api/webhook.py` при POST проверять заголовок `X-Telegram-Bot-Api-Secret-Token` (или параметр в URL) против `TELEGRAM_WEBHOOK_SECRET` из env. При несовпадении возвращать 403.
**Зачем:** Защита от случайных или злонамеренных POST не от Telegram.
**Как:** В `do_POST` прочитать заголовок, сравнить с `os.getenv("TELEGRAM_WEBHOOK_SECRET")`, если задан и не совпадает — `send_response(403)` и выйти.

### 8. Health-check endpoint
**Что:** Отдельный маршрут, например `GET /api/health`, который возвращает 200 и минимальный JSON: `{"ok": true, "notion": true/false}` (проверка доступа к Notion — один лёгкий запрос, например `databases.list` с limit=1).
**Зачем:** Мониторинг (UptimeRobot, Vercel Checks): бот жив и Notion доступен.
**Как:** В Vercel — отдельный `api/health.py` или ветка в существующем handler по path.

### 9. Более понятные ошибки при сбое ИИ ✅
**Что:** Если `understand_message` или `classify` вернули `None` (таймаут, 429, невалидный JSON), отвечать пользователю не только «ИИ временно недоступен», но и коротко: «Можешь записать явно: запиши в [категория]: текст».
**Зачем:** Пользователь знает, что делать без чтения /help.
**Реализовано:** В `handlers.py` при `if not intent` выводится: «Запиши явно: запиши в [категория]: текст. Или: удали последнюю, измени последнюю на [текст], найди заметки про [фрагмент]. Команды: /help».

## Более крупные фичи (несколько дней)

### 10. Напоминания по незавершённым задачам
**Что:** Раз в день (например в 9:00 по твоему часовому поясу) бот присылает в Telegram: «У тебя N задач на сегодня с Due Date = сегодня и статус не „Выполнена“. Список: …» со ссылками.
**Зачем:** Не забывать про задачи без ручного открытия Notion.
**Как:** Отдельный cron (Vercel Cron, GitHub Actions, или отдельный воркер), который дергает внутренний endpoint с секретом, тот запрашивает задачи за сегодня из Notion и шлёт сообщение через `bot.send_message(chat_id=TELEGRAM_USER_ID, ...)`. Нужно хранить `chat_id` (сейчас это можно вывести из последнего update или задать в env).

### 11. Резервное копирование / экспорт
**Что:** Команда «экспорт за неделю» или «бэкап задач» — выгрузка последних заметок (или только задач) в JSON/CSV или отправка файлом в Telegram.
**Зачем:** Резервная копия или перенос в другой инструмент.
**Как:** `NotionClient`: метод, который за период собирает страницы из нужных баз; форматирование в файл; `send_document` в Telegram.

### 12. Поддержка нескольких пользователей (опционально)
**Что:** Не только один `TELEGRAM_USER_ID`, а несколько (или привязка по chat_id к разным Notion parent page / интеграциям). Каждый пользователь видит только свои заметки.
**Зачем:** Поделиться ботом с семьёй/командой без общего Notion.
**Как:** Таблица или env: `user_id -> NOTION_PARENT_PAGE_ID, NOTION_API_KEY` (или одна интеграция с разными parent). Инициализация `NotionClient` по пользователю; rate limit и state уже по user_id.

## Качество кода и надёжность

- **Тесты:** Добавить интеграционные тесты с mock Notion API (например `respx`/`responses`) на сценарии: создание заметки, удаление, «выполнено», парсинг ответа ИИ.
- **Типизация:** Везде аннотации для публичных функций и ключевых структур (уже частично есть).
- **Логирование:** Для отладки на Vercel добавить опциональную отправку логов в внешний сервис или структурированный JSON (например `structlog`), не только `print` и `logging`.
